2.3.2 Para instalar R no El Capitan

  1. The other method involves patching a library that is part of RLink.
  2. Open Terminal.app and navigate to SystemFiles/Links/RLink/SystemFiles/Libraries/MacOSX-x86-64 within Mathematica‘s $InstallationDirectory. There will be a file named libjri.jnilib there
  3. Make a backup of this file using cp libjri.jnilib libjri.jnilib.backup
  4. Rewrite the hard-coded location of the R libraries it uses with the following command:
  5. installnametool -change @loaderpath/libR.dylib /Library/Frameworks/R.framework/Resources/lib/libR.dylib libjri.jnilib
  6. This assumes that R is installed in the standard location within /Library/Frameworks.
  7. Re-launch the Mathematica kernel and proceed as described in the article above. Setting DYLDLIBRARYPATH can now be skipped.

2.3.4 Para gerar imagens para a PLOS

Estou preparando um manuscrito para a PLoS e sofri um pouco para acertar a melhor maneira de criar figuras de acordo com as especificações da revista. Então estou postando minha solução aqui, talvez seja útil para outros sofredores: 

FigToPLoS[gr_, size_, resolution_:300] := 
  Rasterize[gr, "Image", RasterSize -> resolution size/2.54, 
   ImageSize -> 28.3465 size];
(* gr - gráfico; size - tamanho em cm; resolution - se em branco, 300 dpi *)

Depois, clique com o botão direito na saída, "Save Image As…". A PLoS sugere figuras em TIFF: antes de salvar, vá em Options e acerte: "Image Encoding": LZW e "Image Resolution": 300. Se preferir, torne essas configurações como padrão, clicando em "Use Settings as Defaults".

2.8.5 Instalar Google Remote Desktop

Etapa 1: fazer o download da Área de trabalho remota do Google Chrome e configurá-la Faça o download do app Área de trabalho remota do Google Chrome.

Etapa 2: instalar o pacote Debian para os componentes de host Faça o download do pacote Debian de 64 bits aqui.

Etapa 3: criar uma sessão de área de trabalho virtual Se você usa o Ubuntu 12.04, pule para a Etapa 4.

Em usr/share/xsessions, procure o arquivo .desktop correspondente ao ambiente de área de trabalho de sua escolha. Por exemplo, o Cinnamon tem um arquivo chamado "cinnamon.desktop" com o seguinte comando: gnome-session –session=cinnamon Crie um arquivo chamado .chrome-remote-desktop-session no diretório principal com o seguinte conteúdo: exec /usr/sbin/lightdm-session "<YOUREXECCOMMAND>" Substitua <YOUREXECCOMMAND> pelo comando encontrado no fim do arquivo .desktop: No exemplo do Cinnamon, o comando correto seria exec /usr/sbin/lightdm-session 'gnome-session –session=cinnamon' Áreas de trabalho Unity exigem alguns códigos adicionais. O comando completo é DESKTOPSESSION=ubuntu XDGCURRENTDESKTOP=Unity XDGRUNTIMEDIR=/run/user/$(id -u) exec /usr/sbin/lightdm-session 'gnome-session –session=ubuntu' Salve o arquivo chamado ".chrome-remote-desktop-session"​. Etapa 4: permitir conexões remotas No computador, abra o Chrome. Na barra de endereço localizada na parte superior da tela, digite chrome://apps e pressione Enter. Clique em Área de trabalho remota do Google Chrome App Área de trabalho remota do Google Chrome. Em "Meus computadores", clique em Primeiros passos. Clique em Ativar conexões remotas. Insira um PIN, digite-o novamente e clique em OK. Dispense a caixa de diálogo de confirmação.
